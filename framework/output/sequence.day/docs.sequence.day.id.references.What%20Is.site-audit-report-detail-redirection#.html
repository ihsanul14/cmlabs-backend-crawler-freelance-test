Detail Laporan Audit - Redirection

Dalam panduan ini, kami akan membahas detail laporan audit redirection URL untuk membantu Anda mengamati daftar tautan pengalihan dari URL yang dipilih. Data dalam laporan audit meliputi kode status HTTP, atribut follow link, link awal, link tujuan, arah tautan, hingga kondisi redirect URL.

Apa itu Redirection URL?​

Redirect URL terjadi saat terdapat dua pengalihan dari URL awal menuju URL tujuan. Misalnya, ketika halaman A dialihkan menuju halaman B kemudian dialihkan menuju halaman C. Jika pada proses pengalihan halaman B dialihkan kembali ke halaman A, maka kondisi tersebut disebut sebagai pengalihan berulang atau redirect loop.

Gambar 1 - Ilustrasi Redirect URL

Cara Akses Laporan Audit Redirection URL​

Klik ikon setting pada salah satu URL dalam tabel Page Details untuk melihat halaman detail laporan audit. Selanjutnya, pilih menu Redirection guna mengakses detail hasil audit redirection link. Kami menyediakan opsi filter dan pencarian data untuk memudahkan Anda dalam menampilkan tautan pengalihan tersebut.

Gambar 2 - Daftar URL pada Detail Isu

Laporan Audit Redirection URL​

Pada laporan berikut, kami menampilkan informasi kode status HTTP dari tiap redirect URL. Adanya tautan pengalihan ditandai dengan kode status HTTP 3xx. Beberapa kode status HTTP yang mungkin terdapat dalam URL Anda, antara lain:

  • Kode Status 300 Multiple Choices: Menunjukkan bahwa request memiliki lebih dari satu kemungkinan respons, pengguna harus memilih salah satunya.
  • Kode Status 301 Moved Permanently: Menandakan bahwa URI dari resource yang diminta telah diubah secara permanen sesuai dengan respons URI yang diberikan.
  • Kode Status 302 Found: Menunjukkan bahwa resource yang diminta telah dipindahkan sementara ke URL yang diberikan.
  • Kode Status 303 See Other: Menunjukkan bahwa pengalihan tidak menautkan ke resource yang diminta, melainkan ke halaman lain.
  • Kode Status 304 Not Modified: Menunjukkan bahwa resource belum dimodifikasi sejak terakhir kali halaman tersebut diakses.
  • Kode Status 305 Use Proxy: Menandakan bahwa resource yang diminta hanya tersedia melalui proxy, alamatnya tersedia dalam respons.
  • Kode Status 307 Temporary Redirect: Menunjukkan bahwa sumber daya yang diminta telah dipindahkan sementara ke URL yang diberikan. Perbedaan dengan kode status 302 yaitu kode status 307 menjamin bahwa metode dan isi tidak akan diubah ketika permintaan yang dialihkan dibuat.
  • Kode Status 308 Permanent Redirect: Menunjukkan bahwa sumber daya yang diminta telah dipindahkan secara definitif ke URL yang diberikan. Perbedaan dengan kode status 301 yaitu kode status 308 menjamin metode permintaan dan isi tidak akan diubah.

Selain itu, kami pun menyajikan informasi mengenai atribut follow link. Tautan dengan atribut rel dofollow mengizinkan crawler untuk mengikuti link dan mencapai halaman web yang dituju link tersebut. Sementara itu, tautan dengan atribut rel nofollow menunjukkan tautan tidak perlu diindeks oleh crawler karena konten di dalamnya tidak terlalu penting.

Tersedia pula informasi link from untuk mengetahui URL absolute dari halaman mana tautan tersebut berasal. Kami juga menyediakan informasi link to guna memuat URL mana yang akan dituju selanjutnya oleh tautan tersebut.

Anda juga akan melihat informasi direction tiap link. Nilai dari direction dapat berupa internal apabila menuju pada halaman dalam situs Anda atau bernilai eksternal apabila menuju halaman di luar situs Anda. Dalam laporan redirection juga terdapat informasi broken link yang mengindikasikan apakah tautan mengarahkan pengguna pada halaman rusak.

Kami juga mengidentifikasi apakah URL yang diaudit termasuk ke dalam tautan pengalihan berulang atau redirect loop. Redirect loop dapat terjadi karena adanya pengalihan antara dua link. Sebagai contoh, URL 1 mengalihkan menuju URL 2, namun pada URL 2 kembali dialihkan menuju URL 1.

Hal tersebut membuat crawler tidak dapat mengakses dan mengindeks halaman tujuan sehingga akan mempengaruhi posisi halaman pada hasil penelusuran. Oleh karena itu, jika pada laporan audit data loop bernilai true, Anda harus segera memperbaiki tautan pengalihan tersebut untuk menghindari terjadinya masalah pada situs web Anda.

Gambar 3 - Detail laporan audit redirection links

Pengaruh Redirect Chain pada SEO​

Pemilik web membutuhkan laporan ini untuk mengetahui seberapa banyak tautan pengalihan pada tiap URL. Terlalu banyak pengalihan halaman dapat membuat pemuatan halaman berjalan lebih lama serta membuat proses crawling tertunda. Selain itu, redirect chain juga mengurangi jumlah otoritas halaman yang diteruskan.

Google hanya akan mengikuti hingga 5 hops dalam redirect chain tiap satu kali crawl. Jika terdapat lebih dari 5 hops dalam redirect URL Anda, maka dapat menyebabkan Google menghentikan proses crawl dan gagal mengindeksURL tujuan. Hal tersebut akan berdampak buruk terhadap kualitas SEO dan peringkat situs web pada hasil penelusuran.

Demikian pembahasan mengenai detail laporan audit redirection URL. Lakukan optimasi situs web Anda dengan memperbaiki struktur redirect chain dan pastikan tidak ada pengalihan tautan menuju broken link. Jika Anda memerlukan bantuan kami, silahkan hubungi kami melalui support@sequence.day
